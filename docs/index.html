Task 1 — Operation Night Watch

Audio · continual learning · on Kaggle

A 16-class audio classifier (AST) must learn 13 new classes without forgetting the old 16. Score = ½·old accuracy + ½·new accuracy — forgetting costs as much as failing.

  • The whole game: replay mixing (new classes have only 24–60 clips each)
  • ⚠️ Label noise in classes 3/7 (cow vs sheep) & 11/15 (thunderstorm vs rain)
  • ⚠️ Kaggle data has no val split — predict on submission.csv paths
  • Community score to beat: ~87%
▸ Drill card ready — run end-to-end on Kaggle ✅ val 93.5% · public LB 0.808

Task 2 — Robot Delivery Academy

Behavioral cloning · on Kaggle

A robot on an 8×8 grid learns pick-up-and-deliver from expert demonstrations only (observation → action). Officially supervised learning — not RL.

  • Rare actions (pickup/dropoff) under-learned → class weighting
  • 🚫 No search/planning, no extra expert trajectories, no frame stacking

🛠 Recovery moves

  1. Any error → new message: paste the FULL traceback + the failing cell. "Fix this. Code only."

  3. CUDA out of memory → "Rewrite with batch size 4 and gradient accumulation of 2. Code only."
  4. Too slow for 10 min → "Freeze all encoder layers except the last 2 blocks. Code only."
  5. Score too low → paste P4 numbers: "Old acc X, new acc Y. Give me ONE change to improve the weaker side. Code only."
  6. Undefined variable in a snippet → "Define X too. Code only." (Gemma sometimes assumes processor/train_df exist.)

  • Free-tier API/chat sometimes 500s — just resend, it's not your prompt.

Write a PyTorch nn.Module named CNNActionModel whose forward takes (grid, vector): two Conv2d layers on the (6, 8, 8) grid (6->32 then 32->64 channels, kernel 3, padding 1, ReLU after each), flatten, concatenate the 13-feature vector, then Linear to 128 with ReLU and Linear to 6 action logits. First collect all action labels by iterating grid_dataset (the fourth element of each sample), use torch.bincount to count the 6 actions, and build class weights = 1/counts, normalized, for nn.CrossEntropyLoss so rare pickup/dropoff actions are not under-learned. Then from the failed results in cnn_eval['results'] print how many have no action 4 in r['actions'] (failed before pickup) and how many do (failed after pickup). Code only, max 25 lines. Reply with ONLY a single python code block and nothing else — no explanation, no reasoning, no plan, no draft version, no self-check, no line counting. Write the final code directly.

Code only, max 30 lines. Reply with ONLY a single python code block — no explanation, no reasoning, no draft versions, no bullet points.

⚠️ Known traps — don't burn time here

  • No BFS/A*, ever — even "just for data". Greedy walk-toward-target heuristics off the obs direction features = violation (some suspicious 99%+ LB scores likely do this — don't chase them).

Checkpointed precompute (start it, walk away — ~20–45 min on T4, saves to Drive every 25 animals)

I defined `QUESTIONS = questions_pool[:120]` and I have the `predict_answers(animal, questions)` function from before. P3

The solver: greedy questions + soft weights (never hard-eliminate — the oracle is sometimes wrong)

Answer immediately with code — do not deliberate. I have two globals in memory: `QUESTIONS` (a 120-question subset of questions_pool) and `table` (numpy 0/1 array of shape (len(animals_pool), len(QUESTIONS)): rows = animals in `animals_pool` order, columns = QUESTIONS, values = the oracle's predicted answers). Write class `MySolution` with `__init__(self, animals_pool, questions_pool)` and `solve(self, interactor)` implementing exactly this:
1. weights w = numpy ones(len(animals_pool)).
2. While `interactor.remaining_budget()` > 4 and best weight < 0.9 * w.sum(): loop j over range(len(QUESTIONS)) (the global QUESTIONS, NOT questions_pool) and pick the unasked j minimizing abs(weighted yes-fraction of table[:, j] - 0.5); ans = 1 if interactor.ask(QUESTIONS[j]) == 'yes' else 0; w = w * where(table[:, j] == ans, 0.9, 0.1). Never zero out a weight — the oracle is sometimes wrong.
3. Then guess animals in descending weight order until `interactor.is_done()`.
Code only, max 30 lines. Reply with ONLY a single python code block — no explanation, no reasoning, no bullet points.
P4

Free sanity check — NOT a prompt, paste this cell as-is (Gemma reliably choked on this ask — 3 empty replies)

import numpy as np
rng = np.random.default_rng(0)
counts = []
for i in rng.integers(0, len(animals_pool), size=40):
    w = np.ones(len(animals_pool)); asked = set(); c = 0
    for _ in range(15):
        j = min((q for q in range(len(QUESTIONS)) if q not in asked),
                key=lambda q: abs((w * table[:, q]).sum() / w.sum() - 0.5))
        asked.add(j); c += 1
        w *= np.where(table[:, j] == table[i, j], 0.9, 0.1)
        if np.argmax(w) == i: break
    counts.append(c)
counts = np.array(counts)
print(f"mean {counts.mean():.1f}  max {counts.max()}  <=11 questions: {(counts <= 11).mean():.0%}")
